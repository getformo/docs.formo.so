## Autocapture

Autocapture helps you automatically capture events, interactions, and attribution on your onchain app.

Once you install the Formo SDK, **it automatically collects and tracks user [events](/data/events/overview)** (including page views, wallet connects, and transactions) without the need for manual tracking or custom code.

It's the easiest way to capture the user interactions on your site. No need to rely on manual tracking. No need to choose what to measure. No need to wrangle engineers to write code.

## Cross-subdomain tracking

Formo automatically tracks visitors across all subdomains of your main domain, giving you a unified view of user journeys and attribution accuracy. This means you can follow users as they move between your marketing site, app, blog, and other subdomains—no extra setup required.

- **Unified visitor journey**: Track users as they navigate between your main site, app, and other subdomains
- **Accurate attribution**: Attribute conversions to the correct marketing channel, even if users move between subdomains
- **Simplified reporting**: View combined traffic, user behavior, and conversion data in a single dashboard

To enable cross-subdomain analytics, simply use your root domain when adding your project in the Formo dashboard. The Formo SDK handles identity resolution and session management for you, ensuring seamless analytics across your entire product.

## How to set up product analytics

Get started with Formo Product Analytics in under 5 minutes. This guide walks you through installation, verification, and viewing your first insights.

### Step 1: Create a project

1. Go to the [Formo Dashboard](https://app.formo.so) and sign in
2. Click **New Project** in the top navigation
3. Enter your project name and root domain (e.g., `yourdapp.xyz`)
4. Click **Create Project**

<Tip>Use your root domain without `https://` or `www.` to enable cross-subdomain tracking automatically.</Tip>

### Step 2: Install the Formo SDK

Copy your Project ID from the project settings page, then install the SDK using your preferred method.

<Tabs>
  <Tab title="Wagmi (Recommended)">
    If your app uses Wagmi for wallet connections, this is the easiest setup:

    ```bash
    npm install @formo/analytics
    ```

    ```tsx
    import { FormoProvider } from '@formo/analytics';
    import { WagmiProvider } from 'wagmi';

    function App() {
      return (
        <WagmiProvider config={config}>
          <FormoProvider projectId="YOUR_PROJECT_ID">
            <YourApp />
          </FormoProvider>
        </WagmiProvider>
      );
    }
    ```
  </Tab>
  <Tab title="HTML Snippet">
    Add this snippet before the closing `</head>` tag:

    ```html
    <script
      defer
      data-project-id="YOUR_PROJECT_ID"
      src="https://cdn.formo.so/v1/tracker.js"
    ></script>
    ```
  </Tab>
</Tabs>

For more installation options, see the [Installation Guide](/install).

### Step 3: Verify data is flowing

1. Visit your app in a browser
2. Connect a wallet and perform a few actions
3. Return to the Formo Dashboard and open the **Activity** tab
4. You should see your page views and wallet connects appearing in real-time

Once data is flowing, explore the key sections of your analytics dashboard:

| Section | What you'll see |

| **Overview** | Visitors, wallets, transactions, and revenue at a glance |
| **Activity** | Real-time event stream with filters and Live View |
| **Users** | User profiles with wallet labels and segments |
| **Dashboards** | Custom charts and visualizations |
| **Explore** | SQL query editor for custom analysis |
