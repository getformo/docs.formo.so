## Installation

[Install](/install) the JavaScript SDK via CDN or NPM.

```bash
npm install @formo/analytics --save
```

```tsx
// App.tsx (or App.js)

  import React from 'react';
  import ReactDOM from 'react-dom/client';
  import { FormoAnalyticsProvider } from '@formo/analytics';
  import App from './App';

  const root = ReactDOM.createRoot(document.getElementById('root') as HTMLElement);

  root.render(
    <React.StrictMode>
      <FormoAnalyticsProvider writeKey="<YOUR_WRITE_KEY>">
        <App />
      </FormoAnalyticsProvider> 
    </React.StrictMode>
  );
```

```tsx
// Usage on a page component

import React, { useEffect } from 'react';
  import { useFormo } from '@formo/analytics';

  const HomePage = () => {
    const analytics = useFormo();

    useEffect(() => {
      // Track a custom event
      analytics.track('custom_event', { points: 100 });
    }, [analytics]);

    return <div>Welcome to the Home Page!</div>;
  };

  export default HomePage;
```

## Autocapture

The Web SDK automatically captures the events such as page views and wallet events (connect, disconnect, signature, transaction, etc) automatically with full attribution (referrer, UTM, referrals.)

## Identify wallets

As a best practice, make sure [identify](/data/events/identify) is called at the start of every session or page load for logged-in users, if possible.

Call `identify()` after a user connects their wallet or signs in on your website or app:

```ts
analytics.identify({ address }, {
  "name": "Alex Smith",
  "email": "alex@example.com"
});

OR

window.formo.identify();
```

If no parameters are specified, the Formo SDK will attempt to auto-identify the wallet address. Note that most wallets block wallet fingerprinting by default.

## Track custom events

Record [custom user actions](/data/events/track) with the `track()` function:


```ts
analytics.track(
  "Liquidity Added",  // event name
  {
    pool_id: "LINK/ETH",
    revenue: 99.99,     // revenue or fees
    currency: "USD",    // currency
    points: 99.99        // points for XP or rewards
  }
)

OR 

window.formo.track()
```

## API

The Web3 SDK implements the standard [Events API](/data/events#events-api).

## Configuration

### Batching

To support high-performance environments, the SDK sends events in batches. 

```tsx
<AnalyticsProvider
    writeKey={WRITE_KEY}
    options={{
        flushAt: 20, // defaults to batches of 20 events
        flushInterval: 1000 * 60, // defaults to 1 min
    }}
>
```

Customize this behavior with the `flushAt` and `flushInterval` configuration parameters. 

### Debugging

Control the level of logs the SDK prints to the console with the following logLevel settings:

```tsx
<AnalyticsProvider
    writeKey={WRITE_KEY}
    options={{
        logger: {
            enabled: true,
            levels: ["error", "warn", "info"],
        },
    }}
>
```

The SDK omits logs in localhost by default. To enable logs, set `trackLocalhost` to `true`:

```tsx
<AnalyticsProvider
    writeKey={WRITE_KEY}
    options={{
        trackLocalhost: true,
        logger: {
            enabled: true,
            levels: ["error", "warn", "info"],
        },
    }}
>
```

| Log Level | Description |
|-----------|-------------|
| trace | Shows the most detailed diagnostic information, useful for tracing program execution flow. |
| debug | Shows all messages, including function context information for each public method the SDK invokes. |
| info | Shows informative messages about normal application operation. |
| warn | Default. Shows error and warning messages. |
| error | Shows error messages only. |
