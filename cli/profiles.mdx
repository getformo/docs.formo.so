The `profiles` command group lets you query wallet profile data from the Formo API. You can look up individual wallets by address or ENS name, or search across all profiles with filters and sorting.

All profile commands require the `profiles:read` scope on your API key.

## Get a profile

Fetch a single wallet profile by its address or ENS name.

```bash
formo profiles get <address>
```

### Arguments

| Argument | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `address` | string | Yes | Wallet address (`0x...`) or ENS name (e.g. `vitalik.eth`) |

### Options

| Option | Type | Description |
| :--- | :--- | :--- |
| `--expand` | string | Comma-separated list of fields to expand: `apps`, `chains`, `tokens`, `labels` |

### Examples

```bash
# Basic profile lookup
formo profiles get 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045

# Look up by ENS name
formo profiles get vitalik.eth

# Include expanded chain and label data
formo profiles get 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 --expand labels,chains
```

## Search profiles

Search wallet profiles with optional filtering, sorting, and pagination.

```bash
formo profiles search [options]
```

### Options

| Option | Type | Description |
| :--- | :--- | :--- |
| `--address` | string | Filter by a specific wallet address |
| `--limit` | number | Maximum number of results to return |
| `--offset` | number | Pagination offset |
| `--orderBy` | enum | Field to sort by (see table below) |
| `--orderDir` | `asc` \| `desc` | Sort direction |
| `--expand` | string | Comma-separated fields to expand |
| `--conditions` | string | JSON array of `FilterCondition` objects for advanced filtering |
| `--logic` | `and` \| `or` | Logic operator for combining conditions (default: `and`) |

### Sort fields

The `--orderBy` option accepts the following values:

| Value | Description |
| :--- | :--- |
| `last_onchain` | Most recent onchain activity |
| `first_onchain` | Earliest onchain activity |
| `net_worth_usd` | Total wallet net worth in USD |
| `updated_at` | Last profile update time |
| `tx_count` | Total transaction count |
| `first_seen` | First seen in your project |
| `last_seen` | Last seen in your project |
| `num_sessions` | Number of sessions on your app |
| `revenue` | Revenue generated |
| `volume` | Trading volume |
| `points` | Points balance |

### Examples

```bash
# List first 10 profiles
formo profiles search --limit 10

# Top 5 wallets by net worth
formo profiles search --orderBy net_worth_usd --orderDir desc --limit 5

# Search with a condition: net worth greater than $10,000
formo profiles search --conditions '[{"field":"net_worth_usd","op":"gt","value":10000}]' --limit 20

# Combine conditions with OR logic
formo profiles search \
  --conditions '[{"field":"net_worth_usd","op":"gt","value":10000},{"field":"tx_count","op":"gt","value":50}]' \
  --logic or \
  --limit 20
```

## Filter conditions

The `--conditions` option accepts a JSON array of `FilterCondition` objects. Each condition has three fields:

| Field | Type | Description |
| :--- | :--- | :--- |
| `field` | string | The profile field to filter on (e.g. `net_worth_usd`, `tx_count`) |
| `op` | string | The comparison operator |
| `value` | string \| number | The value to compare against |

### Supported operators

| Operator | Description |
| :--- | :--- |
| `eq` | Equal to |
| `neq` | Not equal to |
| `gt` | Greater than |
| `gte` | Greater than or equal to |
| `lt` | Less than |
| `lte` | Less than or equal to |
| `in` | In a list of values |
| `nin` | Not in a list of values |

### Condition examples

```bash
# Wallets with more than 100 transactions
formo profiles search --conditions '[{"field":"tx_count","op":"gt","value":100}]'

# Wallets seen in the last 7 days with high net worth
formo profiles search \
  --conditions '[{"field":"net_worth_usd","op":"gte","value":50000}]' \
  --orderBy last_seen --orderDir desc
```

<Tip>
Pipe the output to `jq` for quick formatting or field extraction:

```bash
formo profiles search --limit 5 --orderBy net_worth_usd --orderDir desc | jq '.[].address'
```
</Tip>
